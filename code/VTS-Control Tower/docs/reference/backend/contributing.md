# Contributing Guide

## Coding standards
- TypeScript, strict mode
- Prefer explicit types on DTOs and entity fields
- Keep controllers thin; move logic to services
- Use `@nestjs/config` for config access

## Commit guidelines
- Use imperative present tense
- Example: `add trip detection rules`

## Branch naming
- `feature/<short-name>`
- `fix/<issue-id>`

## Pull requests
- Describe changes and testing steps
- Link related issues
- Add migration notes if schema changes
